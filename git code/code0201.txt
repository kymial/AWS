01/31

clear : 터미널 화면 청소
pwd : 터미널의 현재 경로(위치)를표시
ls: 현재 폴더의 파일 혹은 폴더리스트를 보여줌
	- : 옵션 추가 명령어
		a : 모든 파일 및 폴더를 보여준다.
		l : 파일 혹은 폴더의 상세정보를 보여준다.
			- / rwx (1) / rwx (2) / rwx (3)
			- d는 폴더, l 바로가기, - 파일
			- 1  rwx : 소유자에 대한 권한 
			- 2. rwx : 소유그룹에 대한 권한
			- 3. rwx : 기타 타인
			- r : 읽기 권한 w : 작성 권한 x : 실행 권한
cd : 디렉토리 폴더 이동
/ : 현재 위치
. : 뒤


02/01

cd /(드라이브명) : (드라이브 이름)으로 이동
cd / (폴더명) /(폴더명) : 한번에 이동하는 명령어
mkrdir (폴더명) : 폴더 생성하는 명령어
rm (파일명, 폴더명) : (파일명, 폴더명) 삭제 명령어
	r 옵션 : 폴더 안에 있는 하위 폴더와 파일까지 모두 삭제
vim(vi) (파일명(확장자포함)) : vim editor로 파일이 존재하지않으면 생성 존재할경우 실행
	- i를 누르면 insert모드(수정모드)
	- : 눌러서 명령어 모드로 진입해서 w 입력하면 저장
	- : 눌러서 명령어 모드로 진입해서 q 입력하면 종료
	- : 눌러서 명령어 모드로 진입해서 wq 입력하면 저장 후 종료
	
vim editor 실행 직후 읽기모드임
		   -> 쓰기모드로 변경하기 위해서는 I or A키를 입력하면 쓰기모드로 변경 가능
		   -> 쓰기모드에서 읽기 읽기모드로 변경할때 esc
		   -> 읽기모드에서 파일 저장 및 종료하기 위해서는
			  : w - 저장/ :q - 종료
			  : wq - 저장 후 종료 / :q! - 저장하지 않고 종료 (강제종료)
			  
cat (파일명) : 내용 미리보기

git config --global user.name : 깃에 누가 등록되었는지 확인하는 명령어
git config --global user.email : 깃에 누구 이메일로 등록되었는지 확인하는 명령어

git remote add 지포지토리이름(깃허브닉네임) 지포지토리주소(깃허브주소)

git push 리포지토지이름 브랜치명

git push -u 리포지토리이름 브랜치명
			: 다음 푸쉬부터는 반드시 지정한 리포지토리이름 브랜치명으로 푸쉬함
			-> git push만 입력해도 해당 리포지토리이름 브랜치명으로 푸쉬됨
			
git clone 리포지토리주소 : 깃 허브에 있는 걸 복제해온다.			
			
git pull 리포지토리이름 브랜치명 : 외부리포지토리 변경 상태를 받아옴(단, 반드시 로컬 프랜치가 커밋이 되어있는 상태)
